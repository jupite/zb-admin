## 2. 核心架构分析

### 2.1 技术栈架构

| 层级 | 技术选型 | 用途 |
|------|---------|------|
| **框架** | Vue 3.2 + TypeScript | 核心框架，组合式API |
| **构建工具** | Vite 3.0 | 快速开发服务器，生产打包 |
| **UI组件库** | Element Plus 2.2 | 企业级UI组件 |
| **状态管理** | Pinia 2.0 + persistedstate | 全局状态管理，持久化 |
| **路由** | Vue Router 4.1 | 单页应用路由 |
| **HTTP** | Axios 0.27 | 网络请求 |
| **图表** | ECharts 5.3 | 数据可视化 |

### 2.2 路由架构设计

**文件**: `src/routers/index.ts`

```typescript
// 路由分层设计
constantRoutes    // 常量路由（登录、404等）
asyncRoutes       // 异步路由（权限控制）
notFoundRouter    // 404兜底路由
```

**权限控制流程**:
```
登录 → 获取Token → 获取用户角色 → 过滤异步路由 → 动态添加路由 → 访问页面
```

**路由模块划分**:
- `table.ts` - 表格相关路由
- `form.ts` - 表单相关路由
- `echarts.ts` - 图表展示路由
- `system.ts` - 系统管理路由
- `dataScreen.ts` - 数据大屏路由
- `excel.ts` - Excel操作路由
- `nested.ts` - 嵌套菜单路由

### 2.3 状态管理架构 (Pinia)

**文件**: `src/store/modules/`

| Store模块 | 功能 | 持久化 |
|-----------|------|--------|
| `user.ts` | 用户信息、Token、角色 | localStorage |
| `permission.ts` | 路由权限、缓存路由 | - |
| `setting.ts` | 主题配置、布局设置 | - |
| `tagsView.ts` | 标签页状态 | - |

## 9. 总结

### 项目优势

1. **现代化技术栈**: Vue 3 + TypeScript + Vite，开发体验优秀
2. **完善的布局系统**: 三种布局模式，响应式设计
3. **丰富的组件库**: 封装了多种业务组件，开箱即用
4. **强大的主题系统**: 支持暗黑模式、主题色切换、灰度/色弱模式
5. **规范的工程化**: ESLint + Prettier + Commitlint + Husky
6. **权限控制完善**: 基于角色的动态路由权限
7. **功能丰富**: 数据大屏、Excel处理、多种编辑器、图表可视化

### 适用场景

- 企业级中后台管理系统
- 数据可视化平台
- 管理系统模板/脚手架
- 学习Vue 3 + TypeScript的示例项目
